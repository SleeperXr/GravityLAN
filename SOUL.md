# GravityLAN Soul

Lightweight, practical homelab network radar and dashboard.

## Core Philosophy

### 1. Homelab-first, always
GravityLAN is designed for trusted home networks and self-hosted lab environments.
That means:
- Usability matters.
- Setup should stay simple.
- Features should solve real homelab problems.

### 2. No enterprise drama
GravityLAN should not require:
- Kubernetes.
- Distributed infrastructure.
- Mandatory external services.
- Complicated deployment choreography.

### 3. Honest security
GravityLAN must be honest about its security model.
- Intended for homelab use.
- Do not expose it directly to the public internet.
- Use VPN, reverse proxy, or network segmentation.

### 4. Lightweight does not mean sloppy
- Readable code.
- Explicit configuration.
- Safe defaults.

### 5. Preserve the project’s spirit
- A sharp homelab tool.
- A trustworthy self-hosted utility.
- Understandable and maintainable.

---

## Product Boundaries
### Focus on:
- LAN discovery.
- Device organization.
- Topology modeling.
- Lightweight agents.

### Avoid:
- Enterprise SIEM scope.
- Full RMM scope.
- Cloud control plane architecture.
- Premature plugin ecosystems.

---

## Engineering Principles
1. **Prefer clarity over cleverness.**
2. **Prefer incremental improvement over giant rewrites.**
3. **Consistency is a feature.**
4. **Safe defaults matter.**
5. **Minimize blast radius.**
